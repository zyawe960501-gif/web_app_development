# 產品需求文件 (PRD)

**專案名稱**：撲克牌桌遊網站
**專案描述**：這是一個線上撲克牌桌遊網站，提供玩家連線對戰、自動分配對手與發牌功能，讓大家能隨時隨地享受撲克牌的樂趣。

---

## 1. 專案概述

### 背景與動機
許多人喜歡玩撲克牌，但實體聚會不易。市面上雖然有許多博弈類遊戲，但缺乏純粹休閒、能和朋友或隨機玩家輕鬆連線的輕量級網頁撲克牌遊戲平台。因此，我們希望打造一個簡單易用的線上撲克牌桌遊網站，降低遊玩門檻。

### 目標用戶
- **所有人**：包含喜歡玩撲克牌的玩家、想打發時間的網友，以及想和朋友線上同樂的群體。

### 核心價值主張
提供簡單、快速、免繁瑣註冊流程的線上撲克牌對戰體驗。系統提供自動媒合對手、公平發牌與即時連線機制，讓玩家能專注於遊戲本身的樂趣。

---

## 2. 功能需求

### 1. 分配對手 (Matchmaking)
- **使用者故事**：作為 **玩家**，我希望 **能快速隨機匹配到其他正在等候的玩家**，以便 **能立刻開始一局遊戲**。

### 2. 發牌系統 (Dealing System)
- **使用者故事**：作為 **玩家**，我希望 **系統能公平地洗牌與隨機發牌**，以便 **確保遊戲的公正性與隨機性**。

### 3. 與對手連線 (Real-time Connection)
- **使用者故事**：作為 **玩家**，我希望 **能在遊戲中與對手即時同步出牌與操作狀態**，以便 **獲得流暢的對戰體驗，不會有資訊落差**。

### 4. 遊戲房間管理 (Room Management)
- **使用者故事**：作為 **玩家**，我希望 **能建立專屬房間並產生邀請連結**，以便 **邀請認識的朋友一起加入同桌遊玩**。

### 5. 遊戲狀態與結算 (Game State & Scoring)
- **使用者故事**：作為 **玩家**，我希望 **系統能依照撲克牌規則自動判斷輸贏並結算畫面**，以便 **省去人工計分的麻煩並清楚知道遊戲結果**。

---

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Flask 進行 API 開發與後端邏輯處理。
- **前端渲染**：使用 Jinja2 模板結合 HTML/CSS/JavaScript。
- **資料庫**：使用 SQLite 儲存使用者、房間及遊戲紀錄。
- **連線技術**：為達成「與對手即時連線」，需使用 WebSocket (如 Flask-SocketIO) 或類似的即時雙向通訊技術。

### 效能與安全考量
- **連線穩定性**：確保出牌動作能低延遲同步到所有玩家畫面。
- **防作弊機制**：玩家的底牌資訊只能由該玩家的客戶端取得，禁止透過 API 或前端代碼看穿對手手牌。
- **資料安全**：防範基本的 SQL Injection 與 XSS 攻擊。

---

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須有 - 核心發布)
- 玩家能設定簡單暱稱進入大廳（免註冊）。
- 建立遊戲房間與加入房間的機制。
- 即時連線機制，雙方能在同一房間內看到彼此。
- 系統自動洗牌、發牌給房間內的玩家。
- 最基礎的一種撲克牌玩法邏輯（例如：抽鬼牌 或 比大小）。

### Should Have (應該有 - 重要優化)
- 隨機分配對手（點擊「尋找對手」自動加入空房間）。
- 遊戲結束後的勝負判定與重新開始按鈕。
- 基本的遊戲音效或出牌動畫。

### Nice to Have (可以有 - 未來擴充)
- 會員註冊與登入系統。
- 歷史戰績、勝率統計與排行榜。
- 遊戲內文字聊天或表情符號互動。
- 更多不同的撲克牌玩法（如大老二、德州撲克等）。
