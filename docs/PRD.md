# PRD - 線上桌遊系統

## 專案名稱
線上桌遊系統 (Online Board Game System)

## 專案描述
這是一個讓使用者可以隨時隨地在線上與朋友或隨機對手進行桌遊對戰的平台。系統提供即時配對、發牌與獎懲等核心機制，滿足玩家遠距娛樂與社交互動的需求。

---

## 1. 專案概述

### 背景與動機
實體桌遊需要玩家群聚且準備繁複的道具，經常受到地理、時間或找不到玩伴的限制。透過線上桌遊平台，可以打破這些限制，讓玩家就算在家也能輕鬆享受桌遊帶來的樂趣。

### 目標用戶
- **實體桌遊愛好者：** 但平時難以湊齊實體玩伴的玩家。
- **休閒娛樂玩家：** 為了尋求短暫放鬆、消磨時間的學生或上班族。
- **線上交友族群：** 喜歡透過遊戲互動來認識新朋友的玩家。

### 核心價值主張
打破空間限制，提供即時、順暢且富含互動性（包含勝負獎懲、即時連線）的線上桌遊體驗，讓遊戲的樂趣垂手可得。

---

## 2. 功能需求

以下列出系統的主要功能及對應的使用者故事：

### 1. 遊戲房間與對手配對 (配對對手)
- **使用者故事：** 作為「玩家」，我希望「能快速隨機配對對手或建立專屬的私人房間」，以便「能自由選擇與陌生人切磋或與好友同樂」。

### 2. 核心遊戲與發牌機制 (發牌)
- **使用者故事：** 作為「玩家」，我希望「系統能自動、公平且隨機地處理洗牌與發牌」，以便「確保遊戲競技的公正性，並省去手動操作的麻煩」。

### 3. 獎懲與結算系統 (獎懲機制)
- **使用者故事：** 作為「玩家」，我希望「在遊戲結束後能有明確的勝負判定及獎賞/懲罰（例如：贏家獲得積分或代幣，輸家扣除積分）」，以便「增加遊戲的刺激感與挑戰成就感」。

### 4. 即時互動與聊天室
- **使用者故事：** 作為「玩家」，我希望「能在遊玩過程中透過文字或內建的表情符號與對手互動」，以便「在線上也能體會到桌遊常見的『嘴砲』與社交樂趣」。

### 5. 個人戰績與歷史記錄
- **使用者故事：** 作為「玩家」，我希望「能查看自己的勝率、過往對戰紀錄與目前積分」，以便「了解自己的實力變化並獲得進步的動力」。

## 3. 非功能需求

### 技術限制
- **後端：** 使用 Flask 框架處理商業邏輯與 API。
- **前端：** 使用 Jinja2 模板引擎與原生 HTML/CSS/JavaScript 進行介面開發。
- **即時通訊：** 考量到發牌與配對的即時性，需評估使用 WebSockets (如 Flask-SocketIO) 或頻繁的 AJAX Polling。
- **資料庫：** 使用 SQLite 儲存使用者資料、積分、對戰紀錄與遊戲房間資訊。

### 效能與安全考量
- **連線穩定性：** 需要妥善處理玩家突然斷線的狀況（保留遊戲狀態或給予適當懲罰）。
- **資訊安全：** 使用者密碼必須加密儲存（如採用 bcrypt），保障個人資料安全。
- **防作弊機制：** 發牌邏輯與勝負驗證必須完全由後端控制，嚴禁將敏感狀態暴露於前端程式碼中。

## 4. MVP 範圍

為了快速驗證市場並上線，我們將功能分為三個開發階段：

### Must Have (核心必備 —— 基礎上線版本)
- 使用者註冊與登入功能。
- 建立公開/私人房間與基本加入功能。
- 單一款桌遊的基礎核心邏輯（建立對戰回合、後端自動發牌）。
- 遊戲勝負判定基礎機制。

### Should Have (應該要有 —— 提升體驗)
- 隨即配對系統。
- 基礎獎懲機制（積分加減計算）。
- 房間內的即時文字聊天。

### Nice to Have (有了更好 —— 擴充功能)
- 全服排行榜系統。
- 特殊稱號或虛擬道具系統（進階獎勵）。
- 支援斷線重連。
- 語音通話功能。
